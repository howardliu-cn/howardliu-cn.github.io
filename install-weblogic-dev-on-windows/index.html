Windows下安装Weblogic Dev版本

2015-03-24
阅读量 0

在开发过程中，有一种情况是比较尴尬的：需要用到某个软件，但是只需要用到这个软件中的一小部分功能，这样既浪费的磁盘空间，又浪费了CPU空间。比如Weblogic，完整版包括很多功能，但是开发过程中，只是需要创建数据源、部署项目，然后进行测试即可。所以开发过程中，需要一款简化的Weblogic（Weblogic开发版）是挺有必要的（如果电脑配置比较壕，也可以任性的使用完整版）。

1 选择合适版本的Weblogic

从Oracle官网的中间件下载页面中选择合适的版本下载。

我下载的是wls1213_devzip_update1，就是weblogic12c的1.3.0小版本中的开发版的第一次更新的zip包。

2 开始安装

2.1 设置环境变量

首先设置环境变量：

  • JAVA_HOME：指定JDK的安装目录，我的是C:\Program Files\Java\jdk1.7.0_75
  • MW_HOME：指定weblogic server的目录，我的是D:\Develop\Weblogic\wls12130
  • JAVA_VENDOR：指定虚拟机类型，Sun或者Oracle，Sun表示使用Sun的HotSpot虚拟机，Oracle表示使用Oracle的JRockit虚拟机。
  • VM_TYPE：指定虚拟机类型

注意：JAVA_VENDOR与VM_TYPE，在最新的wls12130中，可以不用设置，默认为Sun和HotSpot，如下所示：

@rem Set VM_TYPE and JAVA_VENDOR if not set
IF (%VM_TYPE%)==() set VM_TYPE=HotSpot
IF (%JAVA_VENDOR%)==() set JAVA_VENDOR=Sun

2.2 执行configure.cmd

直接在cmd命令窗口中运行%MW_HOME%目录下的configure.cmd文件，这个过程会解压一些jar文件，知道提示是否配置domain：

2.3 创建domain

下面两种方法都能够创建weblogic的域domain，只是默认的名称不同，第一种字符界面创建的默认域叫做mydomain，第二种图形界面创建的默认域叫做base_domain，不止何故。（按照DRY原则，两种方式应该调用同一部分的创建代码，也就不会产生不同的名称，但是看结果是这里用了两套代码。）

1、可以在上面执行configure.cmd命令时，输入Y，然后会出现输入用户名、密码的提示：

Enter username to boot WebLogic server:
Enter password to boot WebLogic server:
For confirmation, please re-enter password required to boot WebLogic server:

注意：密码必须至少包含 8 个字母数字字符, 且至少包含一个数字或一个特殊字符，否则会报错。

2、可以通过执行%MW_HOME%\oracle_common\common\bin下的config.cmd，调用图形界面来创建domain：

根据提示输入相应的配置即可，一般来说，只需要在需要输入用户名、密码的地方输入。

其他地方可以一路“下一步”，然后点击“创建”。

等待domain创建成功。

等到提示配置成功，点击完成，然后启动weblogic server即可。

2.4 启动server

通过domain目录（我的是D:\Develop\Weblogic\wls12130\user_projects\domains\base_domain）中的startWeblogic.cmd启动，这个过程没有报错，且最后出现 Server state changed to RUNNING 字样，说明启动成功。

可以通过http://localhost:7001/console，进入控制台，开始使用weblogic server。
